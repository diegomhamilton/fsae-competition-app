# User Story: FSAE Judge During Inspection Events

## Persona
- **Role:** FSAE Judge
- **Goal:** Execute inspection events efficiently, consistently, and with clear records of pass/fail outcomes.

## Scenario
During inspection events, judges need to evaluate multiple teams in sequence, verify checklist compliance, and submit results quickly while preserving evidence and notes for any failures or rechecks.

## User Story
As an **FSAE judge**, I want to **run inspection checklists by event stage, record step-by-step outcomes, and track active discussions/history**, so that **teams can be assessed fairly and progress through technical inspection with clear feedback**.

## Acceptance Criteria (Draft)
1. Judge can open an inspection event and view the ordered checklist for the selected stage.
2. Judge can mark each step as pass/fail and add notes when required.
3. Judge can submit and persist the stage result for the current team.
